# Gemini Code Review Log - 2026-02-16

## Summary
Performed code review of Swift files in `Sources/` and `Tests/` focusing on Real-time Safety, Code Quality, and Bug Risks.

## Findings

### Critical Issues
1.  **SynthEngine.swift**: `NSLock` usage in `drainMIDI` (Audio Thread) causes Priority Inversion risk.
2.  **SynthEngine.swift**: `midiEvents` array modification in `drainMIDI` causes Memory Allocation/Deallocation on Audio Thread.

### High Priority Issues
1.  **SnapshotRing.swift**: `popLatest()` releases `SynthParamSnapshot` on Audio Thread. Since `SynthParamSnapshot` contains Copy-on-Write Arrays, this triggers memory deallocation on the audio thread.
2.  **VoiceMixer.swift**: `accumulateVoice` allocates memory internally. While currently unused in the hot path, it poses a significant risk if adopted.
