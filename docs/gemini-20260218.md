# Code Review Report - 2026-02-18

## 概要
DEXEDとの波形一致およびリアルタイム安全性を目的とした変更のレビューを行いました。
対象は `M2DX-Core` 内の `Exp2Table`, `DX7Voice`, `Algorithm` および新規追加された `DX7Ref` (DEXED移植コード) とそのテストです。

## 評価サマリ

| 項目 | 評価 | コメント |
| :--- | :--- | :--- |
| **Integer Overflow Safety** | ✅ 合格 | `Exp2Table.swift` 等で `&*`, `&+` を使用し、C言語の `int32_t` オーバーフロー挙動をSwiftで正確に再現しています。 |
| **Real-time Safety** | ✅ 合格 | `DX7Voice.swift` のゲイン補間計算から除算が排除され、ビットシフトに変更されたことで決定論的な実行時間が保証されています。 |
| **DEXED Compatibility** | ✅ 合格 | `dx7ref.c` はDEXEDのロジックを忠実に移植しており、テストスイートによる比較検証も網羅的です。 |
| **Test Coverage** | ✅ 合格 | 基本波形、アルゴリズム、フィードバック、EGスケーリングなど主要な要素がテストされています。 |

## 詳細分析

### 1. `Sources/M2DXCore/Tables/Exp2Table.swift`
- **変更点:** `exp2LookupQ24` 関数内の演算を `Int32` のラッピング演算 (`&*`, `&+`) に変更。
- **分析:** DEXED (C++) では `int` (通常32bit) のオーバーフローは未定義動作ですが、多くの環境でラップすることを利用しています。Swiftでは標準演算子がオーバーフローでトラップするため、`&*` などのラッピング演算子の使用は必須であり、適切に実装されています。これによりDEXEDとのビット完全一致が可能になります。

### 2. `Sources/M2DXCore/Engine/DX7Voice.swift`
- **変更点:** `computeMod`, `computePure`, `computeFb` における `dgain` 計算を除算から右シフト (`>> kLgBlockSize`) に変更。
- **分析:**
  - 除算 (`/`) はCPUサイクル数が可変長になりうるため、リアルタイムオーディオ処理では避けるべきです。
  - `kLgBlockSize` (6) を用いた算術右シフトへの変更は、DEXEDの実装 (`>> LG_N`) と一致しており、かつ高速・一定時間で完了するため適切です。
  - `fbBuf` の更新ロジックもDEXEDと一致しており、フィードバックループの挙動が保証されています。

### 3. `Sources/M2DXCore/Engine/Algorithm.swift`
- **変更点:** `kLgBlockSize` 定数の追加。
- **分析:** 変更は軽微かつ適切です。

### 4. `Sources/DX7Ref/*` (Reference Implementation)
- **分析:**
  - DEXEDの `fm_core.cc`, `dx7note.cc`, `env.cc` 等から主要なロジックがC言語として移植されています。
  - `dx7ref_osc_freq` における周波数計算（粗調・微調・デチューン）の複雑なロジックも再現されており、周波数精度の検証に役立ちます。
  - プロダクションコードには含まれないテスト用ターゲットとしての分離も適切です。

### 5. `Tests/M2DXCoreTests/VoiceComparisonTests.swift`
- **分析:**
  - `compareVoices` ヘルパーを用いて、サンプル単位での完全一致を確認している点が非常に強力です。
  - 156バイトのパッチデータ生成 (`buildPatch156`) もパラメータを細かく制御できており、エッジケース（フィードバック、ベロシティ感度、レートスケーリング）のテストが網羅されています。
  - テストケースが「基本波形」から「複合アルゴリズム」へと段階的に構成されており、回帰テストとして優秀です。

## 結論
変更内容は高品質であり、プロジェクトの目標（DEXED互換、リアルタイム安全性）に合致しています。マージを推奨します。
